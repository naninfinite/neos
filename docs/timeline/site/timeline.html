Stage 1A

First Shell UI Pass

This milestone finalized the navigation boundary model between the site shell and the ME.EXE OS environment.

The top-level site now uses a global channel bar and HOME hub panels, while the taskbar and launcher exist only inside ME.EXE.

In plain English, this made the hierarchy clear: the site is multi-channel, and the OS simulation is contained where it belongs.

Important code

Scoped Shell Composition

export function SiteShell(): ReactElement {
  return (
    <main className="neosShell">
      <div className="wallpaper" aria-hidden="true" />
      <ChannelBar activeChannel={activeChannel} onNavigate={setActiveChannel} />
      <div className="channelViewport">
        <ChannelView channel={activeChannel} onNavigate={setActiveChannel} />
      </div>
      {bootVisible ? <BootOverlay /> : null}
    </main>
  );
}

This separates global channel navigation from local in-channel navigation, so ME.EXE can own the taskbar without imposing it everywhere.
