Stage 1A

First Shell UI Pass

This milestone moved NEOS from launcher-first landing to a desktop-first landing model.

The desktop now shows immediate preview panels for ME.EXE, YOU.EXE, THIRD.EXE, and HOME, with the launcher kept as a secondary path.

In plain English, this changed the first impression from empty space to visible project surfaces right on arrival.

Important code

Desktop-first shell composition

export function DesktopShell(): ReactElement {
  return (
    <main className="neosShell">

}

This keeps runtime scope narrow while exposing core project surfaces directly on the desktop.
