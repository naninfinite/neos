# Task Template

Use this structure for major implementation tasks.

## Title

Short task name.

## Goal

What should exist after this task is complete.

## Authority docs

List the docs that matter for this task.

Example:
- `docs/spec/05-implementation-decisions-lock.md`
- `docs/spec/03-type-contracts.md`
- `docs/spec/02-runtime-architecture.md`

## Constraints

Rules that must not be broken.

Example:
- no legacy imports
- no architecture drift
- no guessing on unresolved behaviour
- small, reviewable changes only

## Deliverables

Concrete output expected from the task.

Example:
- files created
- files updated
- working UI state
- tests added
- timeline updated if visible UI changed

## Validation

How success should be checked.

Example:
- app runs
- tests pass
- typecheck passes
- Gemini confirms alignment with spec

## Timeline impact

State one of:
- no timeline update needed
- timeline update required
